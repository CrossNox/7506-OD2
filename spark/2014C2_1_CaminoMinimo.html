Se tiene un grafo dirigido representado mediante una lista de adyacencias. Dado un nodo inicial y un nodo final queremos que programe usando Map-Reduce un algoritmo que indique cul es el camino mnimo entre ambos nodos. En clase vimos como calcular la longitud, ahora queremos saber cul es el camino.

Al usar Map-Reduce no podemos hacer early-stopping si ya llegamos al destino, ya que para ello deberamos hacer .collect del RDD en cada iteracin. Esto es una mala prctica, ya que el collect es una operacin cara, lenta y que necesita mucho trfico de datos de los workers al driver.

Si tenemos un grafo $G = (V,E)$, entonces necesitamos a lo sumo $|V|-1$ iteraciones, ya que ningn camino ser mas largo que ese valor. Entonces hay que hacer $|V|-1$ iteraciones de Map-Reduce.

Empezamos con un RDD con un registro por cada vrtice, con el siguiente formato: (start, weight_total, adj_list, path)

  • start un vrtice
  • weight_total el costo de llegar a ese vrtice en esa iteracin
  • adj_list lista de adyacencias para ese vrtice

Inicializamos todos los registros con $weight\_total = \infty$ excepto por $start$, con peso $0$.

En la etapa de map, para cada registro en el RDD crearemos una lista con:

  • el registro original
  • una lista de nuevos registros donde start es un elemento de adj_list, weight_total es la suma del peso de ir a ese elemento desde el elemento del registro original y weight_total hasta el momento y adj_list est vaca Luego de explicar la etapa de reduce quedar claro por qu adj_list se crea vaca para esos ltimos registros.

En la etapa de reduce lo que hacemos es simplemente seleccionar el menor valor para weight de los dos nodos que recibimos. adj_list debe ser la lista original para ese nodo. En la etapa de map cuando procesamos ese nodo devolvemos la lista original, todos los dems que devolvemos tienen una lista vaca, asi que bsicamente sumamos todas las listas vacas y la lista original. path ser el valor para path del nodo con el valor mas pequeo para weight.

Encapsulamos toda esa lgica dentro de una funcin que recibe una lista de listas de adyacencias, el vrtice inicial y el final y devuelve un string con el camino y el costo.

Con la lgica explicada previamente en realidad obtenemos el camino minimo del vrtice inicial a todos los demas, pero como solo nos interesa el vrtice final, con filter obtenemos solo ese registro.

def shortest_path(edges,start,end):
    def map_(node):
        lista_nuevos = [node]
        lista_nuevos.extend([(x[0],(node[1][0]+x[1],[],node[1][2]+[x[0]])) for x in node[1][1]])
        return lista_nuevos

    def reduce_(n1,n2):
        weight = n1[0] if n1[0] < n2[0] else n2[0]
        adj = n1[1]+n2[1]
        path = n1[2] if n1[0] < n2[0] else n2[2]
        return (weight,adj,path)
    
    sc.broadcast(end)
    rdd = sc.parallelize([(edgelist[0],(0 if edgelist[0]==start else np.inf,edgelist[1],[edgelist[0]])) for edgelist in edges],8)
    #start, weight_total, adj_list, path
    for _ in range(len(edges)-1):
        rdd = rdd.flatMap(map_).reduceByKey(reduce_)
    rdd = rdd.filter(lambda x: x[0]==end)
    tup = rdd.collect()[0]
    return "No hay camino" if tup[1][2][0] != start else "{} (cost {})".format(" => ".join(tup[1][2]),tup[1][0])
